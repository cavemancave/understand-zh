前言（Preface）

原文版本：Preface

Linux 的开发更强调实践而非理论。当有人提出新算法或对现有实现的修改建议时，通常会被要求提供相应的代码来支撑论点。虚拟内存（VM）系统中使用的许多算法最初由理论研究者设计，但其实现已经与理论产生了相当大的偏离。在某种程度上，Linux 确实遵循了从设计到实现的传统开发周期，但更常见的情况是：根据系统在"真实世界"中的表现以及开发者的直觉判断来进行修改。

这意味着 VM 在实践中表现优异，但除了少数网站上的一些不完整概述之外，几乎没有专门针对 VM 的文档可用——当然，包含本书早期草稿的那个网站除外！这导致了一种状况：只有少数核心开发者才能完全理解 VM。新开发者寻找有关其工作机制的信息时，通常被告知"去读源码"，而关于实现的理论基础几乎没有可用的资料。这就要求即使是一个普通的观察者也需要投入大量时间来阅读代码、研究内存管理领域。

本书详细介绍了 2.4.22 版本中实现的 Linux VM，并对 2.6 版本中可期待的内容做了坚实的入门介绍。除了讨论实现细节外，还将介绍其所基于的理论。本书并非一本内存管理理论书籍，但如果预先了解底层基础，通常会更容易理解 VM 为何以特定方式实现。

作为补充，附录包含了对 VM 中相当大比例代码的详细注释。这应能大幅减少开发者或研究人员理解 Linux VM 内部运作所需投入的时间。由于 VM 的实现即使在主要版本之间也往往遵循相似的代码模式，这意味着在深入理解 2.4 VM 的基础上，后来的 2.5 开发版 VM 和最终的 2.6 发布版将在数周内即可解读。

目标读者（The Intended Audience）

任何对 VM（一个核心内核子系统）如何工作感兴趣的人都能在本书中找到许多问题的答案。VM 比任何其他子系统都更能影响操作系统的整体性能。它也是 Linux 中最不被理解、文档最差的子系统之一，部分原因是它确实有太多内容。如果没有先建立对整个 VM 的强大概念模型，就很难孤立地理解代码的各个部分，因此本书旨在在阅读源码之前提供详细的描述。

这些材料应该对有兴趣将 VM 适配到自身需求的新开发者，以及仅仅想了解 VM 如何工作的读者最有价值。它也将有益于希望在与 VM 交互时充分利用它的其他子系统开发者，以及寻找现代操作系统中内存管理实现细节的操作系统研究人员。对于那些只是好奇想多了解一个备受讨论的子系统的读者，他们将找到易于阅读的 VM 功能描述，覆盖所有细节而无需翻阅源代码。

不过，本书假定读者在阅读前至少已读过一本通用操作系统书籍或一本 Linux 内核相关的通用书籍，并具有 C 语言的基本知识。虽然已尽一切努力使材料易于理解，但仍假设读者具备通用操作系统的基础知识。

本书概览（Book Overview）

在第一章中，我们详细介绍了如何管理和解读源代码。将介绍三个用于分析、便捷浏览和管理代码的工具。主要工具是 Linux 交叉引用（LXR）工具——它允许以网页形式浏览源代码；以及 CodeViz——用于生成调用图，它是在研究本书期间开发的。最后一个工具 PatchSet 用于管理内核和应用补丁。手动应用补丁可能非常耗时，而使用诸如 CVS（http://www.cvshome.org/）或 BitKeeper（http://www.bitmover.com）这样的版本控制软件并非总是可行。借助这个工具，一个简单的规范文件即可确定使用什么源码、应用哪些补丁以及使用什么内核配置。

在后续章节中，将详细讨论 Linux VM 实现的各个部分，例如如何以体系结构无关的方式描述内存、进程如何管理其内存、特定分配器如何工作等等。每章都会引用最接近描述 Linux 行为的论文，并深入介绍实现细节、使用的函数及其调用图，使读者能清晰了解代码的组织结构。每章末尾会有一个"2.6 中的新变化"小节，介绍在 2.6 VM 中可期待的内容。

附录是对 VM 相当大比例代码的注释。它逐行描述了 VM 中一些更复杂的方面。VM 的代码风格即使在内核的主要版本之间也往往保持相当一致，因此对 2.4 VM 的深入理解将是理解 2.6 内核发布时的宝贵助力。

2.6 中的新变化（What's New in 2.6）

在撰写本文时，2.6.0-test4 刚刚发布，因此 2.6.0-final 预计"随时"发布，即 2003 年 12 月或 2004 年初。幸运的是，2.6 VM 在大多数方面与 2.4 相比仍然相当容易辨认。然而，2.6 中确实有一些新材料和概念，忽略它们未免可惜，因此本书增设了"2.6 中的新变化"部分。在某种程度上，这些部分假设你已阅读了本书的其余内容，因此在初次阅读时只需浏览一下即可。如果你决定开始阅读 2.5 和 2.6 的 VM 代码，"新变化"部分中对预期内容的基本描述应该会极大地帮助你的理解。需要注意的是，这些部分基于 2.6.0-test4 内核，在 2.6 发布前应该不会有重大变化。但由于仍有可能发生变化，你应将"新变化"部分视为指导方针而非确定事实。

配套光盘（Companion CD）

本书附带一张配套光盘，供安装了 GNU/Linux 的系统使用。按如下方式将光盘挂载到 /cdrom：

root@joshua:/$ mount /dev/cdrom /cdrom -o exec

光盘中包含一份已构建并配置好的 Apache 1.3.27（http://www.apache.org/），但它要求光盘挂载在 /cdrom/。要启动它，运行脚本 /cdrom/start_server。如果没有错误，输出应如下所示：

mel@joshua:~$ /cdrom/start_server
Starting CodeViz Server: done
Starting Apache Server:  done

The URL to access is http://localhost:10080/

如果服务器启动成功，将浏览器指向 http://localhost:10080 即可使用光盘的 Web 服务。光盘包含的一些功能有：

  • 一个 Web 服务器，通过 /cdrom/start_server 启动。启动后访问的 URL 是 http://localhost:10080。已在 Red Hat 7.3 和 Debian Woody 上测试通过；
  • 整本书以 HTML、PDF 和纯文本格式包含在 /cdrom/docs 中。其中包括一个可搜索的函数索引，可查找有注释的函数。如果搜索的函数没有注释，浏览器将自动重定向到 LXR；
  • 借助 LXR 提供了可在网页中浏览的 Linux 2.4.22 源代码；
  • 使用 CodeViz 工具的在线版本生成调用图；
  • 第一章中讨论的 VM Regress、CodeViz 和 patchset 软件包可在 /cdrom/software 中获得。还提供了 gcc-3.0.4，因为它是构建 CodeViz 所必需的。

要关闭服务器，运行脚本 /cdrom/stop_server，之后即可卸载光盘。

排版约定（Typographic Conventions）

本文档中使用的约定很简单。新引入的概念以及 URL 使用斜体字体。二进制文件和软件包名称使用粗体。结构体、字段名、编译期定义和变量使用 等宽 字体。在讨论结构体中的字段时，有时会同时包含结构体和字段名，例如 page→list。文件名使用等宽字体，但包含文件使用尖括号，如 <linux/mm.h>，可在内核源码的 include/ 目录中找到。

B.1　初始化各内存区

B.1.1　函数：setup_memory

源码位置：arch/i386/kernel/setup.c

本函数的调用图如图 2.3 所示。该函数会收集必要的信息，提供给引导阶段内存分配器（boot memory allocator）用以完成自身的初始化。整个流程可拆分为若干不同的任务：

  • 找出低端内存（low memory）的起始与结束 PFN（min_low_pfn、max_low_pfn），高端内存（high memory）的起始与结束 PFN（highstart_pfn、highend_pfn），以及系统中最后一个物理页的 PFN（max_pfn）。
  • 初始化 bootmem_data 结构，并声明哪些页面可以被引导阶段内存分配器使用。
  • 把系统可用的所有页面标记为“空闲（free）”，然后保留那些用于存放表示页面状态位图（bitmap）所占用的页面。
  • 保留 SMP 配置使用的页面，以及 initrd 镜像（若存在）所占用的页面。

B.1.2　函数：zone_sizes_init

源码位置：arch/i386/mm/init.c

这是用于初始化每个内存区（zone）的顶层函数。各 zone 以 PFN 为单位的大小已在 setup_memory()（参见 B.1.1 节）中确定。该函数填充一个 zone 大小数组，并将其传递给 free_area_init()。

B.1.4　函数：free_area_init_node

源码位置：mm/numa.c

该函数有两个版本。第一个版本几乎与 free_area_init() 完全相同，只是采用了不同的起始物理地址。它适用于那些只有一个节点（因此使用 contig_page_data）但物理地址并非从 0 开始的架构。

B.1.6　函数：build_zonelists

源码位置：mm/page_alloc.c

该函数为请求节点中的每个 zone 构建一份回退 zone 列表。这适用于某次分配在当前 zone 无法满足、需要去咨询其他 zone 的情形。完成之后，从 ZONE_HIGHMEM 的分配会回退到 ZONE_NORMAL；从 ZONE_NORMAL 的分配会回退到 ZONE_DMA；而 ZONE_DMA 则没有可回退的目标。

590 {
591     int i, j, k;
592 
593     for (i = 0; i <= GFP_ZONEMASK; i++) {
594         zonelist_t *zonelist;
595         zone_t *zone;
596 
597         zonelist = pgdat->node_zonelists + i;
598         memset(zonelist, 0, sizeof(*zonelist));
599 
600         j = 0;
601         k = ZONE_NORMAL;
602         if (i & __GFP_HIGHMEM)
603             k = ZONE_HIGHMEM;
604         if (i & __GFP_DMA)
605             k = ZONE_DMA;
606 
607         switch (k) {
608             default:
609                 BUG();
610             /*
611              * fallthrough:
612              */
613             case ZONE_HIGHMEM:
614                 zone = pgdat->node_zones + ZONE_HIGHMEM;
615                 if (zone->size) {
616 #ifndef CONFIG_HIGHMEM
617                     BUG();
618 #endif
619                     zonelist->zones[j++] = zone;
620                 }
621             case ZONE_NORMAL:
622                 zone = pgdat->node_zones + ZONE_NORMAL;
623                 if (zone->size)
624                     zonelist->zones[j++] = zone;
625             case ZONE_DMA:
626                 zone = pgdat->node_zones + ZONE_DMA;
627                 if (zone->size)
628                     zonelist->zones[j++] = zone;
629         }
630         zonelist->zones[j++] = NULL;
631     } 
632 }

597
取出当前 zone 的 zonelist 并清零。
600
把 j 初始化为 0，对应 ZONE_DMA。
601-605
把 k 设为当前正在考察的 zone 类型。

取得 ZONE_HIGHMEM。
615-620

该函数尝试为页面加锁。如果页面无法加锁，则会让当前进程睡眠，直到该页面可用为止。

B.2.1.2　函数：__lock_page

源码位置：mm/filemap.c

该函数在 TryLockPage() 失败后被调用。它会定位该页面对应的等待队列，并在其上睡眠直到能成功获得锁。

897 static void __lock_page(struct page *page)
898 {
899     wait_queue_head_t *waitqueue = page_waitqueue(page);
900     struct task_struct *tsk = current;
901     DECLARE_WAITQUEUE(wait, tsk);
902 
903     add_wait_queue_exclusive(waitqueue, &wait);
904     for (;;) {
905         set_task_state(tsk, TASK_UNINTERRUPTIBLE);
906         if (PageLocked(page)) {
907             sync_page(page);
908             schedule();
909         }
910         if (!TryLockPage(page))
911             break;
912     }
913     __set_task_state(tsk, TASK_RUNNING);
914     remove_wait_queue(waitqueue, &wait);
915 }
899

900-901

903
把当前进程加入 page_waitqueue() 返回的等待队列中。
904-912

905

906

865 }

C.1　页表初始化

C.1.1　函数：paging_init

源码位置：arch/i386/mm/init.c

这是从 setup_arch() 调用的顶层函数。当本函数返回时，页表已经被完整地建立起来。注意这一切都是 x86 特有的实现。

C.1.2　函数：pagetable_init

源码位置：arch/i386/mm/init.c

本函数的职责是从一个名为 swapper_pg_dir 的静态定义 PGD 出发，对页表进行静态初始化。至少要保证有 PTE 指向 ZONE_NORMAL 中的每一个物理页帧。

这第一段代码用来初始化 PGD。具体做法是让每个表项都指向全局的零页（global zero page）。后续真正用于引用 ZONE_NORMAL 中可用内存的表项将稍后再分配。

217

219

220-223
如果启用了 PAE，仅仅把每个表项保留为 0（这等价于让它指向全局零页）是不够的，因为这种情况下每个 pgd_t 都是一个结构体。必须对每个 pgd_t 调用 set_pgd，才能让该表项指向全局零页。
224
把 i 初始化为 PGD 中对应 PAGE_OFFSET 的偏移。换句话说，本函数只会初始化线性地址空间中属于内核的部分，用户空间部分维持原样。
225

这个循环开始建立可用的 PMD 表项。在 PAE 情形下，使用 alloc_bootmem_low_pages() 来分配页面，并相应地设置 PGD。在没有 PAE 时，并不存在中间目录，因此只是把它“折叠（fold）”回 PGD，以保留三级页表的假象。

227

228
计算该 PGD 对应的虚拟地址。
229-230
如果已经到达 ZONE_NORMAL 的末尾，则跳出循环，因为不再需要更多的页表项了。
231-234
如果启用了 PAE，则为 PMD 分配一页，并通过 set_pgd() 将其设置进去。
235
如果没有 PAE，则把 pmd 直接置为当前的 pgd_t。这就是为了模拟三级页表所采用的“折叠回（folding back）”技巧。
237-238
做一次合理性检查（sanity check），确保 PMD 是有效的。

初始化 PMD 中的每一个表项。这个循环只有在启用了 PAE 的情况下才会真正执行多次。要记得，在没有 PAE 时 PTRS_PER_PMD 等于 1。

240
计算这个 PMD 对应的虚拟地址。
241-242
如果已经到达 ZONE_NORMAL 的末尾，则结束。
243-248
如果 CPU 支持 PSE，则使用大 TLB 表项。也就是说，对于内核页面，一条 TLB 项可以映射 4 MiB，而不是通常的 4 KiB，从而第三级 PTE 就不再必要。
258
把 __pe 设置为一个内核页表项的标志位（_KERNPG_TABLE）、表明该表项映射 4 MiB 的标志位（_PAGE_PSE），再加上通过 __pa() 得到的、该虚拟地址对应的物理地址。注意，这意味着有 4 MiB 的物理内存并不是由页表来映射的。
250-253
如果 CPU 支持 PGE，则为该页表项设置 PGE 标志。这会把该表项标记为“全局（global）”，对所有进程可见。
254-255
由于 PSE 使得第三级不再需要，这里直接用 set_pmd() 设置 PMD，然后继续处理下一个 PMD。
258
否则，不支持 PSE，需要 PTE，于是为它们分配一个页面。

260-265

到这一步为止，页表项已经建立完毕，覆盖了 ZONE_NORMAL 的所有部分。剩下需要处理的区域是固定映射（fixed mappings）所用的区域，以及通过 kmap() 映射高端内存页所需的区域。

277
固定地址空间被认为从 FIXADDR_TOP 处开始，并在地址空间中向更低处“结束（finish）”。__fix_to_virt() 接受一个索引作为参数，返回从 FIXADDR_TOP 开始倒数第 index 个页帧（在固定虚拟地址空间内）。__end_of_fixed_addresses 是固定虚拟地址空间所使用的最后一个索引。换言之，这行代码返回的就是与固定虚拟地址空间起始处相对应的 PMD 的虚拟地址。
278

287-290

C.1.3　函数：fixrange_init

源码位置：arch/i386/mm/init.c

本函数为固定的虚拟地址映射建立有效的 PGD 和 PMD。

167 static void __init fixrange_init (unsigned long start, 
                                      unsigned long end, 
                                      pgd_t *pgd_base)
168 {
169     pgd_t *pgd;
170     pmd_t *pmd;
171     pte_t *pte;
172     int i, j;
173     unsigned long vaddr;
174 
175     vaddr = start;
176     i = __pgd_offset(vaddr);
177     j = __pmd_offset(vaddr);
178     pgd = pgd_base + i;
179 
180     for ( ; (i < PTRS_PER_PGD) && (vaddr != end); pgd++, i++) {
181 #if CONFIG_X86_PAE
182         if (pgd_none(*pgd)) {
183             pmd = (pmd_t *) alloc_bootmem_low_pages(PAGE_SIZE);
184             set_pgd(pgd, __pgd(__pa(pmd) + 0x1));
185             if (pmd != pmd_offset(pgd, 0))
186                 printk("PAE BUG #02!\n");
187         }
188         pmd = pmd_offset(pgd, vaddr);
189 #else
190         pmd = (pmd_t *)pgd;
191 #endif
192         for (; (j < PTRS_PER_PMD) && (vaddr != end); pmd++, j++) {
193             if (pmd_none(*pmd)) {
194                 pte = (pte_t *) alloc_bootmem_low_pages(PAGE_SIZE);
195                 set_pmd(pmd, __pmd(_KERNPG_TABLE + __pa(pte)));
196                 if (pte != pte_offset(pmd, 0))
197                     BUG();
198             }
199             vaddr += PMD_SIZE;
200         }
201         j = 0;
202     }
203 }

176

177

178

180

182-187

C.2　页表遍历

C.2.1　函数：follow_page

源码位置：mm/memory.c

本函数返回在 mm 的页表中、位于 address 处的 PTE 所使用的 struct page。

405 static struct page * follow_page(struct mm_struct *mm, 
                                     unsigned long address, 
                                     int write) 
406 {
407     pgd_t *pgd;
408     pmd_t *pmd;
409     pte_t *ptep, pte;
410 
411     pgd = pgd_offset(mm, address);
412     if (pgd_none(*pgd) || pgd_bad(*pgd))
413         goto out;
414 
415     pmd = pmd_offset(pgd, address);
416     if (pmd_none(*pmd) || pmd_bad(*pmd))
417         goto out;
418 
419     ptep = pte_offset(pmd, address);
420     if (!ptep)
421         goto out;
422 
423     pte = *ptep;
424     if (pte_present(pte)) {
425         if (!write ||
426             (pte_write(pte) && pte_dirty(pte)))
427             return pte_page(pte);
428     }
429 
430 out:
431     return 0;
432 }
405
参数依次为：即将被遍历页表的 mm、感兴趣的 struct page 所对应的 address，以及 write，后者用来指示该页是否即将被写入。
411
取得 address 对应的 PGD，并确保它存在且有效。
415-417
取得 address 对应的 PMD，并确保它存在且有效。
419
取得 address 对应的 PTE，并确保它确实存在。
424
如果该 PTE 当前是 present 的（处于存在状态），则我们有可返回的结果。
425-426
如果调用者表明即将进行写入操作，则检查该 PTE 是否设置了写权限；如果有，则将该 PTE 标记为脏页（dirty）。
427
如果 PTE 处于存在状态且权限合适，则返回该 PTE 所映射的 struct page。
431
返回 0，表示该 address 没有关联的 struct page。

附录 J　页框回收

J.1　页缓存操作

本节阐述如何向页缓存与 LRU 链表中添加和删除页，这两者紧密关联。

J.1.1　向页缓存添加页

J.1.1.1　函数：add_to_page_cache

来源： mm/filemap.c

在调用 __add_to_page_cache() 前先获取保护页缓存的锁；该函数会将页加入页哈希表和 inode 队列，以便能快速找到属于某个文件的页。

667 void add_to_page_cache(struct page * page, 
                     struct address_space * mapping,
                      unsigned long offset)
668 {
669       spin_lock(&pagecache_lock);
670       __add_to_page_cache(page, mapping, 
                        offset, page_hash(mapping, offset));
671       spin_unlock(&pagecache_lock);
672       lru_cache_add(page);
673 }
669获取保护页哈希表和 inode 队列的锁
670调用执行“真正”工作的函数
671释放保护哈希表和 inode 队列的锁
672将页加入页缓存。page_hash() 根据 mapping 和文件内的 offset 哈希到页哈希表中。若返回一个页，则发生了冲突，冲突的页通过 page→next_hash 和 page→pprev_hash 字段串成链表

J.1.1.2　函数：add_to_page_cache_unique

来源： mm/filemap.c

该函数在许多方面与 add_to_page_cache() 非常相似。主要区别在于：本函数在向缓存中添加页之前会先持有 pagecache_lock 自旋锁检查页缓存。它供那些可能与其他进程竞争在缓存中插入页的调用者使用，例如 add_to_swap_cache()（见 K.2.1.1).

675 int add_to_page_cache_unique(struct page * page,
676         struct address_space *mapping, unsigned long offset,
677         struct page **hash)
678 {
679     int err;
680     struct page *alias;
681 
682     spin_lock(&pagecache_lock);
683     alias = __find_page_nolock(mapping, offset, *hash);
684 
685     err = 1;
686     if (!alias) {
687         __add_to_page_cache(page,mapping,offset,hash);
688         err = 0;
689     }
690 
691     spin_unlock(&pagecache_lock);
692     if (!err)
693         lru_cache_add(page);
694     return err;
695 }
682获取 pagecache_lock 以检查缓存
683使用 __find_page_nolock() 检查页是否已存在于缓存中（见 J.1.4.3)
686-689如果页不在缓存中，使用 __add_to_page_cache() 添加（见 J.1.1.3)
691释放 pagecache_lock
692-693如果页之前不在页缓存中，使用 lru_cache_add() 将其加入 LRU 链表（见 J.2.1.1)
694若本次调用将页加入了页缓存则返回 0，若页已存在则返回 1

J.1.1.3　函数：__add_to_page_cache

来源： mm/filemap.c

清除所有页标志，将其加锁，取一次引用并将其加入 inode 与哈希队列。

653 static inline void __add_to_page_cache(struct page * page,
654       struct address_space *mapping, unsigned long offset,
655       struct page **hash)
656 {
657       unsigned long flags;
658 
659       flags = page->flags & ~(1 << PG_uptodate |

J.1.1.5　函数：add_page_to_hash_queue

来源： mm/filemap.c

该函数将 page 添加到以 p 为头的哈希桶顶部。要注意 p 是数组 page_hash_table 中的元素.

 71 static void add_page_to_hash_queue(struct page * page, 
                                       struct page **p)
 72 {
 73     struct page *next = *p;
 74 
 75     *p = page;
 76     page->next_hash = next;
 77     page->pprev_hash = p;
 78     if (next)

来源： mm/filemap.c

                               struct address_space *mapping,
973                            unsigned long offset, struct page *hash)
974 {

976

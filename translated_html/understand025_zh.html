H.1.2　计算 Slab 中的对象数量

H.1.2.1函数：kmem_cache_estimate

来源：mm/slab.c

在创建缓存时需要确定一个 slab 中能存储多少对象，以及会有多少浪费。下面这个函数计算 slab 中可存储的对象数，并考虑了 slab 和 bufctl 是否必须存放在 slab 内的情况。

388 static void kmem_cache_estimate (unsigned long gfporder, 
             size_t size,
389          int flags, size_t *left_over, unsigned int *num)
390 {
391     int i;
392     size_t wastage = PAGE_SIZE<<gfporder;
393     size_t extra = 0;
394     size_t base = 0;
395 
396     if (!(flags & CFLGS_OFF_SLAB)) {
397         base = sizeof(slab_t);
398         extra = sizeof(kmem_bufctl_t);
399     }
400     i = 0;
401     while (i*size + L1_CACHE_ALIGN(base+i*extra) <= wastage)
402         i++;
403     if (i > 0)
404         i--;
405 
406     if (i > SLAB_LIMIT)
407         i = SLAB_LIMIT;
408 
409     *num = i;
410     wastage -= i*size;
411     wastage -= L1_CACHE_ALIGN(base+i*extra);
412     *left_over = wastage;
413 }

H.1.3　缓存收缩

kmem_cache_shrink() 的调用图见 图 8.5。提供了两种收缩函数。kmem_cache_shrink() 从 slabs_free 中移除所有 slab，并返回因此释放的页面数。__kmem_cache_shrink() 释放 slabs_free 中所有的 slab，然后验证 slabs_partial 和 slabs_full 是否为空。这在缓存销毁期间很重要，此时关心的不是释放了多少页面，而是缓存是否为空。

H.1.4　缓存销毁

当模块被卸载时，它负责销毁自己创建的所有缓存；因为在模块加载时已确保不会出现两个同名缓存。核心内核代码通常不销毁自己的缓存，因为这些缓存的存在贯穿整个系统的生命周期。销毁缓存的步骤如下：

  • 从缓存链中删除该缓存
  • 收缩缓存以删除所有 slab（见 8.1.8 节）
  • 释放所有 per CPU 缓存（kfree()）
  • 从 cache_cache 中删除缓存描述符（见 8.3.3 节）
